说说代理

正向代理

正向代理是在客户A和服务器B之间加多个代理服务器Z，用户A通过代理服务器Z来请求访问服务器B，客户端必须要进行一些特别的设置才能使用正向代理。如图：

正向代理的作用
  1. 访问本无法访问的服务器B
    假设路由器坏了。。。
  2. 加速访问服务器B
  3. Cache作用
  4. 客户端访问授权
  5. 隐藏访问者的行踪

反向代理

对于客户端而言，代理服务器就像是真的原始服务器，而且客户端不需要进行任何特别的设置。客户端向反向代理的命名空间(name-space)中的内容发送普通请求，接着反向代理将判断向何处(原始服务器)转交请求，并将获得的内容返回给客户端

反向代理的作用
  1. 保护和隐藏原始资源服务器
  2. 负载均衡

透明代理

客户端根本不需要知道有代理服务器的存在,透明代理设备会根据自身的策略拦截并修改用户的请求的报文，并作为实际的请求方向服务器发送请求。

电子作业防抄袭之winnowing算法

算法简介

电子作业防抄袭算法必须满足：

  1. 不被空格、换行符、字母大小写、标点符号等影响
  2. 不被文档中某些字符串的简单调换下顺序影响（position independence）
    k-grams是一种常见的指纹算法，它将一个文档的字符串分割成长度为K的字符串集合（k-grams），相邻的小字符串之间只差一个字符（错位），然后分别计算这些字符串的各个hash值，然后从hash值中选取一段作为文档的指纹fingerprint。

    举个栗子

    A do run run run, a do run run

    过滤

    adorunrunrunadorunrun

进行5-grams

adoru dorun orunr runru
unrun nrunr runru unrun
nruna runad unado nador
adoru dorun orunr runru
unrun

通过一种hash算法分别计算hash值

77 72 42 17 98 50 17 98 8 88 67 39 77 72 42 17 9

接下来最重要的就是sample选取的问题了

  1. 选取所有hash值，计算量(n-k+1)*n
  2. 0 mod p，即选出hash值中mod p为0的值。

    缺点：假设我们有一个文档D，字符数为105，采用k-gram算法且k取值6，则它会产生105-6+1=100个hash值，文档D有5个段落p1, p2, p3, p4, p5，假设使用0 mod p算法，p取值10，那么很有可能这(105-6+1)/10=10个hash值只分布在段落p1, p2, p4中，而p3和p5中没有一个hash值被选取出来。虽然这种概率不大，但是我们不能保证p1~p5这5个段落都有值被选取出来进行比较
